Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: The grid search technique is a technique to select varied sets of hyperparameters which can then be used in a model. The search space, or hyperparameter combinations, are manually specified and searched exhaustively. For example, using a K Nearest Neighbor model, we can create a hyperparameter search space with varying n_neighbors and weights. A model will then be created for each of these combinations. Following this, each of these models will then be fitted and cross-validated. The accuracies of each model with respect to the parameter selection will then be determined as the average of the scores made on each of train/test sets which were created using the K-fold technique. Finally, the model that best fits, or has the highest accuracy score, respective to the parameters will be chosen.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: The k-fold cross-validation training technique is a cross-validation technique which divides a set of data into k-folds where k is chosen by the model creator. Once the data set is split into the k folds, k-1 of the folds are labeled training data while one of the folds is used as a validation set. This labelling of folds is repeated K times so that each of the K data points is used as a validation set once, thus leaving us with K train/test data sets. The model is then trained and validated on each of these sets and each of their scores are averaged to determine a single accuracy score of the model.

The benefits of using the k-fold cross-validation technique while utilizing grid search is that we can check if a model has high variance before using it in real world applications. Additionally, by utilizing the K-folds technique, we are maximizing our use of the data set by training our model on every data point at least once while ensuring that each of the data points is used for validation once.

To demonstrate the effectiveness of the K-folds technique, consider if we had utilized only a single train/test subset (1/2 test, 1/2 train). In this scenario, the cross-validation score will be less insightful and less accurate than if we were to use a similar K-folds set up where K=2. In the latter example, we essentially do the same as the former, only now we cross-validate our model one additional time on a new set of training/testing data. We should expect the 2-fold score to be more accurate due to the additional cross-validation performed.

(There is some discussion online on choosing K and its implications for obtaining a representative CV score. It seems like the general idea is that lower K > 2 will have bias towards higher error scores, while larger K will have lower pessimistic bias, but higher variance and therefore a misleading CV score. Hence, it is better to pick a happy medium relative to the data set size. It seems like most people suggest K=10 or K=5.

Intuitively, it seems like a higher K would always be better, but others have mentioned in link (3), below, that higher K leads to higher variance in the CV scores. The reasoning seems to be that higher K leads to overlapping of training sets. This seems to imply dependant test errors which further imply that there is a higher variance in the CV score. Furthermore, this variance is difficult to capture as stated in the paper by Bengio in link (3). Thus, any model used with a high K risks not capturing the variance in CV scores and ultimately choosing a less good model.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV
    

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 1)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {
        'max_depth' : range(1,11) 
    }


    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    
    # Return the optimal model after fitting the data
    return grid.best_estimator_
